project overview

Linux Log Anomaly Detector

A CLI-based defensive security tool that ingests Linux system logs, applies rule-based detection logic to surface suspicious activity, and generates a structured incident report — demonstrating practical SOC-analyst thinking on a real Fedora system.

Detection rules (MVP)
  • Brute force SSH
    >5 failed logins from same IP within 60s window
  • Sudo anomaly
    sudo usage by account with no prior sudo history
  • New user creation
    useradd / adduser events in auth.log
  • Off-hours login
    successful auth outside configurable work-hours window

SCOPE CONSTRAINT — COMMITTED
No Flask. No real-time streaming. No database. No web interface. If the MVP is done early, one optional extension is permitted — phishing domain lookup against known IOC feeds, or a simple CSV export. The HTML report is the dashboard. That is enough.
